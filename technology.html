서버 클러스터

각기 다른 서버(Server Enterprise or server Datacenter)들을 하나로 묶어서 하나의 시스템같이 동작하게 함으로써
클라이언트들에게 고가용성의 서비스를 제공하는것을 말한다.
클러스터로 묶인 한시스템에 장애가 발생하면, 정보의 제공 포인트는 클러스터로 묶인 다른 정상적인 서버로 이동한다.
서버클러스터는 사용자로 하여금 서버 기반 정보를 지속적이고, 끊기지않게 제공받을수 있게 한다.



서버 클러스터의 구성

클러스터 기술은 세가지 유형의 장애를 대비합니다.

어플리케이션과 서비스 장애
시스템과 하드웨어 장애
여러기관의 사이트 장애
- 어플리케이션과 필수 서비스에 영향을 미치는 경우
- 하드웨어를 구성하는 CPU, drives , memory, network adapters, 전원공급기에 영향을 미치는 경우
- 자연재해, 정전, 연결 중단 등으로 발생할수 있다


Single Quorum Device Cluster

두 개 이상의 노드가 있고 모든 노드가 하나 이상의 클러스터 저장 장치에 연결되도록 구성된 클러스터 구성입니다.
클러스터 구성 데이터는 단일 클러스터 저장 장치에 저장됩니다.


MNS (Majority Nodes Set)

MNS (Majority Nodes Set)는 전체 서버에 분산된 공유 저장소 버스 및 쿼럼이 있거나 없는 다중 서버 구성입니다.
MNS (주 노드 집합) 클러스터에서 각 노드는 쿼럼 장치 데이터의 로컬 복사본을 유지합니다.
Windows Server 2003 Enterprise는 클러스터당 최대 4개의 노드를 지원하고 Datacenter는 최대 8개의 노드를 지원합니다.
MNS (주 노드 집합) 클러스터의 각 노드는 쿼럼의 로컬 복사본을 유지 관리하므로 공유 저장 장치가 필요하지 않으며 MNS (주 노드 집합) 클러스터는 지리적으로 분산된 환경에서 WAN을 통해 배포할 수 있습니다.


+ Quorum

클러스터의 구성에 필수적으로 항목 or 정책
예를 들어 quorum disk 3이라고 한다면, Disk array에 5개의 Disk중 3개 무조건 활성화 상태여야만 클러스터가 정상 동작중이라고 할 수 있다

Quorum Data를 보관 관점에서 보면 cluster구성을 갖고 있는 Disk어레이(DB)를 하나로 구성할지, 여러개로 구성할지등으로 구분될 수 있다.
대다수의 시스템이 single quorum Data 방식을 따른다
